# Assembler.Resolving

The third stage of the YAML-to-game pipeline. Converts parsed `ValueSource<T>` instances (from `Assembler.Parsing`) into runtime `IValueProvider<T>` instances that behaviours read during gameplay. Also holds all per-run state registries (variables, assets, compiled expressions, entity transforms, exclusive groups).

`ValueResolver.Resolve<T>` is the extension method that dispatches on `ValueSource` subtype and returns an `IValueProvider<T>`: `ValueProvider<T>` for plain mutable values, `ExpressionValueProvider<T>` for compiled expressions, `TriggerOutputProvider<T>` for trigger-supplied values, and `NullValueProvider<T>` for `None<T>` sources. `ResolutionContext` bundles the registries passed to every resolve call. `VariableRegistry` holds globals, `EntityVariableScope` provides per-entity locals that shadow them, `AssetRegistry` loads Unity assets via `Resources.Load`, and `CompiledExpressionsRegistry` caches compiled delegates. The `Behaviours/` subdirectory holds one `BehaviourData` (or `TriggerData`) subclass per behaviour — the resolved data bag passed to each `GameBehaviour<TData>` during two-phase initialisation in `Assembler.Building`.
