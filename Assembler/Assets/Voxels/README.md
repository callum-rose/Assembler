# Assembler.Voxels

LLM-driven voxel generation and I/O. Handles the full lifecycle: prompt an LLM (via `Assembler.Anthropic`) to produce Goxel plain-text, parse it into a model, convert between Y-up and Z-up coordinate systems, encode to MagicaVoxel `.vox` binary, and save assets under `Assets/Resources/Voxels/`.

`VoxelModel` is the core in-memory representation (voxel dictionary + palette + bounds). `GoxelTextParser` / `GoxelTextWriter` handle the Goxel plain-text format, `VoxReader` / `VoxWriter` handle the `.vox` binary format, and `GoxelCoordinateConverter` swaps Y and Z axes (the LLM is prompted to produce Y-up Unity coordinates while `.vox` and Goxel use Z-up). The `Pipeline/` subdirectory provides a fluent async pipeline builder (`VoxelGenerationPipeline`) with composable stages for prompting, refining, axis-swapping, deduping, encoding, and persistence; observer callbacks are marshalled to the Unity main thread via `IMainThreadDispatcher`. The `Editor/` subdirectory exposes the pipeline as the `VoxelGeneratorWindow` Unity Editor tool.
