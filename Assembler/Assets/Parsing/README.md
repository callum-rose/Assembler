# Assembler.Parsing

The second stage of the YAML-to-game pipeline. Transforms raw DTOs (from `Assembler.Deserialisation`) into validated, immutable `Info` records consumed by `Assembler.Building`. All property values are wrapped in `ValueSource<T>` (with subtypes for constants, variable references, expressions, asset refs, trigger outputs, entity positions, template parameters, and "none") so resolution to concrete values is deferred until runtime.

`Transformer.Transform(GameDto)` is the entry point. `GameInfo` is the root record; `BehaviourInfo` and its subtypes (in `Info/Behaviours/`) describe each behaviour declaratively; `ListenerInfo` and its three variants describe trigger wiring (direct, entity-tagged, behaviour-tagged). `BehaviourRegistry.All` is the static catalogue mapping YAML behaviour names to their factory functions and property descriptors — every new behaviour type is registered here. `TemplateInstantiator` expands templates with parameter substitution, and `TransformContext` threads variables, parameters, expressions, and a type registry through the recursive transformation.
